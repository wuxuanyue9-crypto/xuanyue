# 老冯交易系统可编程规则 v0.3

> 目标：把老冯后花园的交易框架转换为可人工执行、可回测、可进一步编码的明确规则。v0.3新增“冰点核心修复先手”和ETF独立通道；原有简单回调、阳阴阳不放宽。

## 0. 使用边界

这是一份研究与回测规格，不是当日荐股，也不直接连接券商下单。

规则中的参数分为两层：

- **【原始口径】**：聊天记录中老冯明确说过的规则或数字。
- **【工程默认】**：为了让程序能够运行而补齐的公式和阈值，必须通过历史样本外回测再决定是否采用。

不能把工程默认写成“老冯固定规则”。尤其要避免把聊天中标明“来源 DeepSeek”的转贴参数当成老冯本人规则。

本版默认：

- 市场：股票主模块覆盖沪深 A 股，包含主板、创业板和科创板；容量占比的分子、分母均暂不纳入北交所。
- ETF：只进入独立研究通道，不与股票混排，也不使用股票“板块前三”定义。
- 频率：日线为主；冰点先手实盘版必须另有1分钟或5分钟快照。
- 信号：交易日 `t` 收盘数据完整后生成。
- 执行：最早在 `t+1` 交易。
- 持股：短线到小波段，常规不超过 5 个交易日。
- 账户：只做研究、筛选、回测和提示，不进行真实自动下单。

当前交易制度下，普通股票买入后在交收前不能卖出，因此回测必须处理 T+1；深交所现行规则同时规定竞价买入数量按 100 股或其整数倍申报。上交所、深交所 2026 年交易规则及其生效日期见官方文件：[上交所规则](https://www.sse.com.cn/lawandrules/sselawsrules2025/trade/universal/c/c_20260424_10816492.shtml)、[深交所规则](https://docs.static.szse.cn/www/lawrules/rule/trade/current/W020260424690713155663.pdf)。

## 1. 一页执行总表

一笔新交易必须依次通过以下六层。策略C允许在板块尚为强势候选时拿修复先手，但仍须通过逻辑、核心身份和专属风控：

```text
市场不是主跌
-> 板块达到主线标准，或达到策略C的ICE_LEADER_CANDIDATE标准
-> 个股通过大成交、高波动、辨识度过滤
-> 简单回调、阳阴阳或冰点核心修复形态成立
-> 止损、目标、R 和仓位可以在买前计算
-> 没有触发“不及预期”否决项
```

| 层级 | 必须回答 | 不合格动作 |
| --- | --- | --- |
| 大盘 | 是否主跌、冰点、修复或高潮？ | 主跌不新开；冰点仅运行预先建池的策略C |
| 板块 | 容量、强度、联动、持续性、逻辑是否合格？ | 低于容量底线或只有孤立个股则放弃 |
| 个股 | 是否高成交、高历史波动、高辨识度核心？ | 小成交、后排、独狼过滤 |
| 形态 | 是否为回调、未完成阳阴阳或冰点核心修复？ | 不把普通弱股大跌误当冰点买点 |
| 风控 | 结构止损、目标、盈亏比和仓位是否明确？ | 任一项无法计算则不买 |
| 预期 | 何时必须出现何种反馈？ | 到期无反馈或相对转弱则主动退出 |

### 1.1 当前校准快照

程序正式运行用最近5个完整交易日的沪深成交额中位数。当前本地只有2026-07-08至07-10三个完整全市场截面，临时中位数为29,135.40亿元，因此研究默认先用：

| 项目 | 临时动态门槛 | 用法 |
| --- | ---: | --- |
| 板块不做线 | 277.70亿元 | 低于直接排除 |
| 板块大板块线 | 457.94亿元 | 达到后才参加主线评估 |
| 工程容量A档 | 300.00亿元 | 独立工程标签，不等于大板块 |
| 个股LOOSE | 15.04亿元 | 只进观察池 |
| 个股STANDARD | 19.96亿元且成交额前6.25% | 正常交易候选默认 |
| 个股STRICT | 26.49亿元且成交额前4.25% | 震荡或冰点修复期 |

如果只按07-10单日33,883.62亿元计算，板块两条线会升至322.96/532.57亿元，个股三档会升至17.49/23.21/30.81亿元。单日值用于压力测试，不替代5日中位数正式口径。

2026-07-08案例回放严格使用截至07-07的五日中位数31,824.52亿元，对应大板块线500.20亿元、个股STRICT线28.93亿元。该案例值只属于历史回放，不覆盖运行时滚动口径。

## 2. 数据定义

### 2.1 两套股票范围

**市场宽度范围 `breadth_universe`**：

- 当日有有效 `close` 和 `pre_close` 的沪深 A 股。
- 包含 ST，用于真实反映全市场涨跌情绪。
- 排除当日停牌和没有可比前收盘价的股票。

**可交易范围 `trade_universe`**：

- 在市场宽度范围基础上排除 ST、退市整理和风险警示股票。
- 排除上市不足 60 个交易日的股票。
- 排除当日停牌股票。
- 排除没有历史板块归属或历史数据不足 60 日的股票。

ST、上市状态、板块成分必须使用当日可知的历史数据，不能用今天的名单回填历史。

**ETF范围 `etf_universe`**：

- 仅纳入境内股票型、行业主题型场内ETF；排除货币、债券、商品、跨境、杠杆和反向产品。
- 至少上市60个交易日，且有完整的基金日线、复权因子和跟踪指数映射。
- ETF与股票分别排名。ETF不能取得“板块个股前三”分，也不能计入板块活跃股票数量。
- `fund_daily`用于真实成交价和成交额，`fund_adj`用于处理拆分等复权事件；技术指标必须基于连续复权价格。

**策略C分钟数据**：

- 市场、板块和标的至少保存09:35、10:30、14:30、14:50快照；正式回测优先使用1分钟数据。
- 每个快照必须带`observed_at`，只允许引用该时点已经形成的最高、最低、VWAP、涨跌家数和排名。
- 只有日线时只能运行保守代理，结果不得与分钟版合并。

### 2.2 基础公式

```text
ret_t           = close_t / pre_close_t - 1
amplitude_t     = (high_t - low_t) / pre_close_t
true_range_t    = max(high_t-low_t,
                      abs(high_t-pre_close_t),
                      abs(low_t-pre_close_t))
ATR14_t         = mean(true_range, 14)
ATR_pct_t       = ATR14_t / close_t

adv_count       = count(ret_t > 0)
dec_count       = count(ret_t < 0)
flat_count      = count(ret_t = 0)
valid_count     = adv_count + dec_count + flat_count
adv_ratio       = adv_count / valid_count
dec_ratio       = dec_count / valid_count
median_ret      = median(ret_t)
market_amount   = sum(amount_t)

rank_pct_desc(x) = rank(x, descending, method="min") / valid_item_count
```

本规则所有`*_rank_pct`都使用`rank_pct_desc`：数值越小排名越靠前；例如`0.06`表示前6%。并列项使用最小名次，回测中不得一处按百分位、一处按名次百分比。

涨跌停数量必须使用当日实际涨跌停价格或行情状态字段，不能把所有股票统一按 10% 判断。

若使用 Tushare，日线字段包括 `open/high/low/close/pre_close/pct_chg/vol/amount`，其中 `amount` 单位为千元，进入本规则前必须乘以 1,000 转为元：[Tushare 日线字段](https://tushare.pro/document/1?doc_id=27)。

### 2.3 四大指数

默认指数：

```yaml
indices:
  - 000001.SH  # 上证指数
  - 399001.SZ  # 深证成指
  - 399006.SZ  # 创业板指
  - 000688.SH  # 科创50
```

### 2.4 成交额归一化口径

板块和个股的绝对成交额必须与同一范围的市场总成交额配套：

```text
market_amount_sh_sz_t = sum(沪深A股 amount_t)
market_amount_ref_t   = median(market_amount_sh_sz, 最近5个已完成交易日)
```

- 盘后筛选：`market_amount_ref_t`可以包含当日。
- 盘中筛选：只使用前一交易日已经完成的5日中位数；没有可靠的全日成交额预测时，不使用当日未收盘数据抬高或降低门槛。
- 历史回测：市场总额、板块额、个股额必须使用同一日期、同一市场范围、同一币种单位。
- 数据不足5日时不生成正式信号；本章后面的2026年示例仅有3个完整本地截面，所以明确标为“临时3日中位数示例”。

运行时统一公式：

```text
dynamic_amount_floor = calibrated_market_share * market_amount_ref_t
```

绝对金额只保留为历史证据和异常数据保护，不再作为跨年度固定门槛。

## 3. 大盘择时规则

老冯对指数的核心要求不是预测涨跌，而是“只要不是主跌就可以交易”；冰点、修复和高潮主要用于调整仓位。

市场状态分成两个互不替代的维度：

```text
trend_state   = UP / RANGE / MAIN_DOWN
emotion_state = ICE / ICE_REPAIR / NORMAL / EUPHORIA
```

### 3.1 主跌定义

单个指数：

```text
bear_index_i = close_i < MA20_i
               AND MA20_i < MA20_i[-5]

bull_index_i = close_i > MA20_i
               AND MA20_i > MA20_i[-5]
```

**【工程默认】主跌状态机：**

```text
main_down_candidate =
  count(bear_index_i) >= 3
  AND mean(adv_ratio, 3) < 40%
  AND mean(median_ret, 3) < 0

进入 MAIN_DOWN：最近 3 日中至少 2 日满足 main_down_candidate

退出 MAIN_DOWN：
  连续 2 日 count(bear_index_i) <= 1
  AND adv_ratio >= 45%

进入 UP：
  count(bull_index_i) >= 3
  AND mean(adv_ratio, 5) >= 50%

其余为 RANGE
```

这套两日确认用于减少状态来回跳变。均线周期和宽度阈值均是工程默认。

### 3.2 冰点定义

**【原始口径的历史锚点】**：

- 4,500 家下跌被称为明显冰点。
- 300 家上涨被称为“冰点开盘”。
- 42 只跌停被称为“差不多冰点”。

上市公司数量会变化，程序用比例而不是只用绝对家数。

**【工程默认】冰点条件：**

```text
ice_breadth = adv_ratio <= 10%
ice_median  = median_ret <= -2.0%
ice_panic   = limit_down_count >= max(40, ceil(0.8% * valid_count))
ice_lack    = limit_up_count <= rolling_quantile(limit_up_count, 60, 10%)

ICE = 上述至少满足 2 项
      AND (ice_breadth OR ice_panic)

EXTREME_ICE = 上述至少满足 3 项
```

`10%`大致把“300 家上涨”转换成可随市场扩容变化的比例；`40 只跌停`来自历史锚点。其余组合方式是工程默认。

冰点不是无条件抄底：

| 组合状态 | 新开仓规则 |
| --- | --- |
| `MAIN_DOWN + ICE` | 所有新策略禁用，包括策略C |
| `RANGE/UP + ICE` | 原策略只做已有计划主线核心；策略C可冻结ICE_LEADER_CANDIDATE，待分钟修复 |
| 单纯“接近冰点” | 不提前猜冰点，继续等收盘或修复 |

### 3.3 冰点修复

老冯曾以“至少修复到上涨 1,500 家以上”作为具体时点判断。程序转换为比例并增加跌停、指数确认。

**【工程默认】**：

```text
ICE_REPAIR =
  最近 2 日内出现 ICE
  AND adv_ratio >= 30%
  AND adv_ratio - ice_day_adv_ratio >= 20 个百分点
  AND median_ret > 0
  AND limit_down_count <= 50% * ice_day_limit_down_count
  AND count(index_ret_i > 0) >= 2
```

冰点修复只恢复到小风险交易，不能直接视为强趋势。

### 3.4 高潮定义

老冯把板块/市场高潮理解为超买：当天让人产生“必须赶紧买”的紧迫感，但第二天可能缺少接盘。

**【工程默认】**：

```text
euphoria_breadth = adv_ratio >= 80%
euphoria_median  = median_ret >= 2.0%
euphoria_limitup = limit_up_count >= rolling_quantile(limit_up_count, 60, 90%)

EUPHORIA = 上述至少满足 2 项
```

高潮动作：

- 禁止因为普涨临时追高开新仓。
- 盈利持仓兑现 25%-50%。
- 等回调，不把高潮日当低风险买点。

### 3.5 市场状态与仓位上限

以下全部是**工程默认**：

| 市场状态 | 总仓位上限 | 单笔账户风险 |
| --- | ---: | ---: |
| `MAIN_DOWN` | 0%-10%，仅处理旧仓 | 0%，不新开 |
| `MAIN_DOWN + ICE` | 0%-10%，仅处理旧仓 | 0%，不新开 |
| `RANGE/UP + ICE_CORE_REPAIR` | 模块上限20% | 基础风险×0.50模块系数 |
| `ICE_REPAIR` | 20%-30% | 0.50% |
| `RANGE` | 20%-40% | 0.50% |
| `UP + MAINLINE` | 40%-60% | 0.50%-1.00% |
| `EUPHORIA` | 降至 30%-50% | 禁止追高新开 |

老冯早期“7.5 成算满仓”只能作为账户上限参考，不代表每个市场状态都应达到 75%。

## 4. 主线板块规则

### 4.1 统一板块层级

板块只能在同一分类、同一层级内比较：

- 申万一级与申万一级比较。
- 申万二级与申万二级比较。
- 同花顺概念与同花顺概念比较。

不能把申万一级行业和细分概念直接混在同一排行榜。

概念板块允许成分重叠，但每个板块的成交额应单独计算；不能把所有板块成交额再相加当作市场总额。

### 4.2 容量门槛

**【原始口径】**：

- 板块成交额低于 100 亿元不做。
- 后期把“有逻辑和资金共振、成交 200 亿元以上”称作大板块。
- 大盘缩量时，成交额过滤网可以相应降低。

**【历史锚点回算】**：

聊天中的100亿元表述发生在2023-08-05周六，回算使用前一交易日2023-08-04；200亿元表述发生在2024-12-26。公开收盘数据核对如下：

| 口径 | 对应交易日 | 当日沪深成交额 | 历史阈值 | 阈值占市场 |
| --- | --- | ---: | ---: | ---: |
| 不做线 | 2023-08-04 | 10,491.60亿元 | 100亿元 | 0.953143% |
| 大板块线 | 2024-12-26 | 12,724.60亿元 | 200亿元 | 1.571759% |

市场数据来源：[2023-08-04沪深成交10,491.60亿元](https://www.stcn.com/article/detail/939672.html)、[2024-12-26沪深成交12,724.6亿元](https://www.stcn.com/article/detail/1469379.html)。

因此程序不再写死100/200亿元，而是保存两个历史占比：

```text
sector_hard_share = 100 / 10491.6 = 0.009531435
sector_big_share  = 200 / 12724.6 = 0.015717586

hard_floor = sector_hard_share * market_amount_ref_t
big_floor  = sector_big_share  * market_amount_ref_t
```

300亿元只保留为**工程A档参考值**，不冒充老冯原话，也不能替代“大板块线”：

```text
engineering_a_floor = max(300亿元, hard_floor)
```

两个标签独立判断：

- `< hard_floor`：直接排除。
- `hard_floor <= sector_amount < big_floor`：容量合格，但还不是老冯后期所说的大板块。
- `>= big_floor`：取得“大板块”标签，可以参加主线评估。
- `>= engineering_a_floor`：取得工程容量A档；A档本身不等于主线，也不自动取得“大板块”标签。

**2026-07-10回算示例**：本地全市场截面扣除北交所后，沪深成交额为33,883.62亿元；2026-07-08至07-10三个完整截面的沪深成交额中位数为29,135.40亿元。完整5日数据补齐前，临时结果为：

| 使用基准 | 不做线 | 大板块线 | 工程A档线 |
| --- | ---: | ---: | ---: |
| 临时3日中位数29,135.40亿元 | 277.70亿元 | 457.94亿元 | 300.00亿元 |
| 07-10单日33,883.62亿元 | 322.96亿元 | 532.57亿元 | 322.96亿元 |

所以在当前高成交环境里，直接沿用100/200亿元会明显放宽标准；300亿元只能是工程标签，不能据此宣称已经达到历史等价的“大板块”。

口径敏感性：若历史与当前都统一改用“沪深京全A”，07-10对应的不做线约324.74亿元、大板块线约530.21亿元；与沪深口径差异不大。补充口径核对参考：[2023-08-04万得全A约10,503亿元](https://www.htfc.com/wz_upload/png_upload/20230804/1691154397480edc6b2.pdf)、[2024-12-26北交所140.83亿元](https://www.stcn.com/article/detail/1468853.html)。程序只能二选一并全程一致，禁止用历史沪深分母配当前全A分母。

### 4.3 当日主线命中

```text
sector_amount_share = sector_amount / market_amount
sector_adv_ratio    = sector_adv_count / sector_valid_count

active_stock =
  stock_ret >= 3%
  AND stock_amount >= LOOSE动态流动性线

active_core_count = count(active_stock)
```

**【工程默认】当日命中：**

```text
sector_day_hit =
  sector_amount >= big_floor
  AND sector_amount_rank_pct <= 20%
  AND sector_return_rank_pct <= 20%
  AND sector_adv_ratio >= 60%
  AND active_core_count >= 3
```

老冯会把成交额榜前 100 和涨幅榜前 100 按板块合并观察；程序增加两个联动字段：

```text
top100_amount_count = 板块进入全市场成交额前100的股票数
top100_gain_count   = 板块进入全市场涨幅前100的股票数

linkage_ok = top100_amount_count >= 3
             AND top100_gain_count >= 3
```

`3 只`是工程默认。原始案例曾出现“成交额榜靠前 4 只铜股、涨幅榜也有多只”，但不是公开的固定三只或四只规则。

### 4.4 主线状态机

```text
WATCH
  -> CANDIDATE：当日 sector_day_hit
  -> MAINLINE：最近 3 日至少 2 日 sector_day_hit
               AND linkage_ok
               AND logic_confirmed = true
  -> WEAKEN：MAINLINE 后首次不满足维持条件
  -> EXIT：连续 2 日不满足维持条件
```

维持条件比进入条件略宽：

```text
sector_amount_rank_pct <= 40%
AND sector_return_rank_pct <= 40%
AND sector_adv_ratio >= 45%
AND sector_amount >= hard_floor
```

主线不是“今天涨就是，明天跌就不是”，因此用状态机避免单日切换。

### 4.5 逻辑字段

价格数据只能识别资金行为，不能自动证明产业逻辑。逻辑必须可追溯：

```yaml
logic_confirmed: true
logic_type: policy | industry | earnings | event
logic_source: "来源名称或本地笔记链接"
logic_source_date: YYYY-MM-DD
logic_expire_date: YYYY-MM-DD
logic_summary: "一句话说明为什么可能持续"
```

没有可追溯逻辑时，板块最多进入 `CANDIDATE`，不能进入正式 `MAINLINE`。

### 4.6 同时跟踪数量

**【原始口径】**：通常同时盯 1-2 条主线、1-2 个轮动板块，共 3-4 个板块、约 50 只自选股。

程序默认：

- 主线最多保留得分前 2。
- 轮动最多保留得分前 2。
- 每个板块保留辨识度最高的 10-15 只。
- 全部候选池最多 50 只。

## 5. 高波动、高辨识度个股规则

### 5.1 硬过滤

**【原始口径】**：

- 选股顺序是“主线板块 -> 个股辨识度 -> 形态”。
- 个股扫描曾用日成交额大于 8 亿元，后期改为大于 6 亿元。
- 看历史波动，不只看一天振幅。
- 不用换手率做硬过滤。

**【历史锚点回算】**：

| 口径 | 对应交易日 | 当日沪深成交额 | 历史阈值 | 阈值占市场 |
| --- | --- | ---: | ---: | ---: |
| 早期扫描 | 2023-05-18 | 8,799.30亿元 | 8亿元 | 0.090916% |
| 后期扫描 | 2024-09-26 | 11,624.35亿元 | 6亿元 | 0.051616% |

市场数据来源：[2023-05-18沪深成交8,799.30亿元](https://www.cnfin.com/yw-lb/detail/20230518/3866042_1.html)、[2024-09-26沪深成交11,624.35亿元](https://stcn.com/article/detail/1333576.html)。

两个历史锚点差异较大，不能只挑一个看起来最合适的数。程序保留三档：

```text
loose_share    = 6 / 11624.35 = 0.000516158
strict_share   = 8 / 8799.3   = 0.000909163
standard_share = sqrt(loose_share * strict_share)
               = 0.000685034

stock_liquidity_floor = selected_share * market_amount_ref_t
```

- `LOOSE`：只进入观察池，不直接取得交易资格。
- `STANDARD`：正常交易候选的默认线。
- `STRICT`：震荡、修复初期或只想保留最强流动性时使用。

**2026-07-10回算示例**：

| 使用基准 | LOOSE | STANDARD | STRICT |
| --- | ---: | ---: | ---: |
| 临时3日中位数29,135.40亿元 | 15.04亿元 | 19.96亿元 | 26.49亿元 |
| 07-10单日33,883.62亿元 | 17.49亿元 | 23.21亿元 | 30.81亿元 |

07-10沪深A股市场额分母共5,194只；排除名称含ST或退市标识后，排名样本4,982只。按单日基准，三档分别约落在成交额前415只（前8.33%）、前303只（前6.08%）、前210只（前4.22%）。这组排名只用于校验当前门槛是否离谱，不能冒充2023/2024历史分位；正式回测还须使用历史时点的风险警示、上市天数和停牌状态重建交易范围。

若全程改用沪深京全A口径，07-10由历史两端反推的个股区间约为17.54亿至30.92亿元；补充口径参考：[2023-05-18北交所24.76亿元](https://www.cnfin.com/gs-lb/detail/20230519/3866548_1.html)、[2024-09-26全A约11,664亿元](https://paper.cnstock.com/html/2024-09/27/content_1972895.htm)。同样不能把全A和沪深口径混用。

**【工程默认】可执行过滤：**

```text
selected_mode =
  STRICT,   if trend_state=RANGE or emotion_state=ICE_REPAIR
  STANDARD, if trend_state=UP and sector_state=MAINLINE
  LOOSE,    only for watchlist

rank_cap = 4.25%, if STRICT
           6.25%, if STANDARD
           8.50%, if LOOSE

liquidity_ok =
  amount_t >= selected_share * market_amount_ref_t
  AND amount_rank_pct <= rank_cap

liquidity_stable = median(amount, 20) >= 50% * stock_liquidity_floor

volatility_ok =
  median(amplitude, 20) >= 5%
  AND 3% <= ATR_pct <= 10%

stock_support = 最近30日、截至当日已确认且不高于close的最高pivot_low
distance_to_support = (close-stock_support) / close

trend_tradable =
  close >= MA20
  AND distance_to_support <= 6%

hard_stock_filter =
  sector_state = MAINLINE
  AND liquidity_ok
  AND volatility_ok
  AND trend_tradable
```

`rank_cap`和50%稳定性线都是工程默认；`liquidity_stable`用于辨识度加分，不作为老冯原始硬过滤。

`WEAKEN` 只用于管理已有持仓，不允许据此新开仓。

`5%`历史振幅、`3%-10%` ATR 和 `6%`止损距离均为工程默认。

### 5.2 辨识度代理分

老冯对辨识度的解释是：板块里的龙一、龙二、龙三；提到板块就会想到它；能领涨、有号召力、交投活跃。

程序用 5 分代理：

| 条件 | 得分 |
| --- | ---: |
| 20 日平均成交额位于板块前 3 或前 20% | 1 |
| 20 日涨幅位于板块前 3 或前 20% | 1 |
| 最近10日至少2天跑赢板块2个百分点，且当天成交额达到LOOSE动态流动性线 | 1 |
| 当前收盘不低于 20 日最高价的 92% | 1 |
| 盘前人工字段 `core_flag=true`，且带确认日期 | 1 |

```text
recognition_pass =
  score >= 3
  AND 成交额条件必须得分
  AND (20日涨幅条件 OR 相对板块强度条件)至少得一分
```

最终每个板块只保留得分最高的 3 只作为交易候选，其余留在观察池。

### 5.3 人工核心字段不能事后回填

```yaml
core_flag: true
core_reason: "板块成交第一/领涨/号召力/事件核心"
core_confirmed_at: YYYY-MM-DD
```

回测只能从 `core_confirmed_at` 当日之后使用该标签，不能在一只股票后来成为龙头后，把它回填到过去。

### 5.4 高波动必须降低风险

**【原始口径】**：高波动低仓位、低波动高仓位。

**【工程默认】风险缩放：**

```text
volatility_multiplier =
  1.00, if ATR_pct < 5%
  0.75, if 5% <= ATR_pct < 7%
  0.50, if ATR_pct >= 7%
```

实际单笔风险：

```text
effective_risk_pct = base_account_risk_pct
                     * market_multiplier
                     * volatility_multiplier
```

## 6. 策略 A：简单回调

### 6.1 老冯原始七条规则

原始聊天里有一套完整规则：

1. 顺势见位交易，即上升趋势中的回调。
2. 时间级别为日线。
3. 持续形态为双重底、三重底。
4. 反转信号为双 K 线反转或内包十字星上破。
5. 前波段低点止损。
6. 前波段高点之下，固定 2 倍盈亏比止盈。
7. 以损定仓，止损金额为账户总额的 1%。

下面只补足程序所需的精确定义。

### 6.2 上升趋势

**【工程默认】**：

```text
uptrend =
  close > MA20 > MA60
  AND MA20 > MA20[-5]
  AND ret_20d > 0
```

不满足上升趋势时，不做“简单回调”；下降趋势里的下跌不叫低吸机会。

### 6.3 已确认摆动低点

以下5根摆动定义、30日窗口和ATR容差均为**工程默认**：

```text
pivot_low[i] =
  low[i] < low[i-1]
  AND low[i] < low[i-2]
  AND low[i] <= low[i+1]
  AND low[i] <= low[i+2]
```

由于需要右侧两根K线，`pivot_low[i]`的`confirmed_at`是`i+2`收盘；在此之前不可使用。

在第一次回踩之前已经确认、且最近30日内最后一个摆动低点定义为水平支撑：

```text
support_id    = pivot日期+价格
support_price = latest_confirmed_pivot_low(asof=first_touch_date-1)
is_touch[d]   = abs(low[d] - support_price) <= 0.5 * ATR14[d]
```

`support_id`、`support_price`和`confirmed_at`在形态开始后冻结，不能随着后来的更低点移动。

### 6.4 双底/三底

**【工程默认】**：

连续贴着支撑的多根K线只能算一次触底。一个新的底必须先离开、再回来：

```text
touch_episode_start[d] =
  is_touch[d]
  AND 距上一episode代表日 >= 3个交易日
  AND 两次episode之间至少有一日 close >= support_price + 1.0*ATR14

bottom_count = count(touch_episode_start) IN {2, 3}
last_touch_age <= 3个交易日
从第一次touch至信号日，不得 close < support_price - 0.5*ATR14

pattern_low = min(low, 从第一次touch至信号日)
```

3日间隔、离开1ATR、最后触底3日和跌破0.5ATR均为工程默认。这样不会把普通横盘里的连续触碰误数成双底/三底。

### 6.5 双 K 反转

老冯只给出“双K反转”类别，以下阳包前阴式定义是**工程默认**：

```text
two_bar_reversal =
  close[-1] < open[-1]
  AND close[0] > open[0]
  AND close[0] > high[-1]
```

`0`为信号日，`-1`为前一交易日。

### 6.6 内包十字星上破

先出现内包十字星；25%实体占比是**工程默认**：

```text
inside_doji =
  high[-1] < high[-2]
  AND low[-1] > low[-2]
  AND abs(close[-1]-open[-1]) / max(high[-1]-low[-1], tick) <= 25%
```

信号日上破：

```text
inside_doji_break = close[0] > high[-1]
```

### 6.7 买入

信号日`t`收盘后冻结所有形态字段：

```text
signal_high = high[t]
signal_low  = low[t]
stop        = pattern_low - max(1 tick, 0.1*ATR14[t])

resistance = buy_trigger上方、截至t收盘已确认、价格最近的pivot_high
buy_trigger = signal_high + 1 tick
R0          = buy_trigger - stop

chase_limit = buy_trigger + 0.5*R0
room_limit  = (resistance - 1 tick + 2*stop) / 3
risk_limit  = stop / (1 - 6%)
buy_limit   = min(chase_limit, room_limit, risk_limit)
```

没有已确认的`resistance`、`R0<=0`、`R0/buy_trigger<2%`或`buy_limit<buy_trigger`，直接取消。`0.1ATR`缓冲、`0.5R0`追高限制和`2%-6%`风险带都是工程默认；先用`R0`消除了“成交价依赖R、R又依赖成交价”的循环引用。

下一交易日`t+1`：

```text
if open[t+1] > buy_limit:
    cancel
elif open[t+1] >= buy_trigger:
    candidate_entry = open[t+1] * (1 + slippage)
elif high[t+1] >= buy_trigger:
    candidate_entry = buy_trigger * (1 + slippage)
else:
    no_fill

if candidate_entry > buy_limit:
    cancel
else:
    entry = candidate_entry
```

订单只在`t+1`有效；所有判断只使用`t+1`当时可见的开盘和成交信息。

### 6.8 止损、止盈和仓位

```text
R_price = entry - frozen_stop
target  = entry + 2*R_price

room_ok = target <= frozen_resistance - 1 tick
risk_ok = 2% <= R_price/entry <= 6%
```

必须 `room_ok AND risk_ok`，否则不交易。

仓位：

```text
risk_budget = equity * effective_risk_pct
raw_shares  = risk_budget / R_price
shares      = floor(raw_shares / 100) * 100

position_value <= 20% * equity
position_value <= remaining_market_exposure
```

参数说明：

- 原始简单回调系统的账户风险为 1%。
- v0.2 小白默认 `base_account_risk_pct=0.5%`，1%只作为上限。
- 单票 20%市值上限是工程默认。
- `2%-6%`风险带、单票20%上限、摆动识别和缓冲均为工程默认；原始七条只有“前波段低点止损、前波段高点之下2R、账户风险1%”。

### 6.9 卖出

- `low<=stop`：记录盘中`HARD_STOP`事件，在最早可卖且可成交时全部退出。
- `high>=target`：记录盘中`TAKE_PROFIT`事件，在最早可卖且可成交时全部退出。
- 第3个可卖交易日收盘，`close/entry-1<3%`：记录`TIME_FAIL`，下一可成交时点退出。
- 第5个可卖交易日收盘仍未退出：记录`MAX_HOLD_EXIT`，下一可成交时点退出。

3日3%和最长5日属于老冯通用退出习惯与本版合并项，不属于“简单回调七条”的组成部分；5日上限是工程默认。

退出按**事件发生时间**而不是静态原因排序。盘中已成交的2R止盈，不能被收盘后才成立的“不及预期”覆盖；受T+1、跌停或停牌阻碍时，首个有效事件进入`EXIT_PENDING`并锁定，直到可成交。

若同一日 K 线同时触及止损和止盈，只有日线数据时按止损先发生处理；有分钟线时才按真实先后顺序。

## 7. 策略 B：阳阴阳

### 7.1 正确定义

阳阴阳不是机械的三根 K 线，而是微型二段式上涨：

```text
单根大实体阳线启动
-> 1至3根小K或窄幅回调
-> 预期第二根等距大阳线
```

买点在中间整理阶段的尾盘，第二根大阳（形态的第三部分）主要是卖点；第二根大阳已经完成后不能追买。

### 7.2 前置过滤

```text
sector_state = MAINLINE
recognition_pass = true
liquidity_ok = true
prelaunch_turnover_ok = true
trend_state != MAIN_DOWN
```

只有主流板块、高辨识度、大成交个股才允许运行阳阴阳。

### 7.3 第一根大阳线

**【原始口径】**：单独一根大实体阳线，最好涨停或接近涨停；实体不够大则放弃，不能把多根阳线合并。

**【工程默认】**：

```text
limit_pct = 当日该股票实际涨停幅度

first_yang =
  ret_day1 >= 70% * limit_pct
  AND (close_day1-open_day1)/pre_close_day1 >= 50% * limit_pct
  AND (close_day1-low_day1) / max(high_day1-low_day1, tick) >= 70%

prelaunch_turnover_ok =
  amount_day1 >= STANDARD动态流动性线
  AND count(amount >= 50% * STANDARD动态流动性线, 前5日) >= 2
```

按实际涨停幅度归一化后，10%股票的首阳最低约7%，20%股票最低约14%，避免把20cm股票的普通7%上涨误当“接近涨停”。比例均为工程默认。

老冯的一般PA表达里有“不单独看量能”的说法，但在阳阴阳语境中也明确排除“前面没有筹码交换、突然20cm涨停”的形态，并认为大成交换手后的阳阴阳更可信。因此本版把前置换手设为工程过滤，不能写成“阳阴阳完全不看量”。

### 7.4 中间整理

**【原始口径】**：

- 可以是一根阴线，也可以多根小 K 或横盘。
- 最多 3 根；出现第 4 根就放弃。
- 中间“阴”的最高价超过前一天才显得强。
- 第一根阳线和中间阴线的最低价应尽可能接近。
- 买点必须等尾盘才能判断形态。

**【工程默认】**：

```text
middle_day_index IN {1, 2, 3}
first_body = close_day1 - open_day1
first_mid  = open_day1 + 0.5*first_body

middle_valid_so_far =
  min(middle_close) >= first_mid
  AND max(middle_high)-min(middle_low) <= 0.6*first_body
  AND max(abs(middle_close-middle_open)) <= 0.4*first_body
  AND min(middle_low) >= first_low * (1 - 0.5%)
  AND current_middle_high >= previous_day_high

low_alignment_score =
  abs(min(middle_low)-first_low) / first_low <= 2%

middle_upper_edge = max(middle_high, 截至当前决策时点)
current_bar_return = current_price / pre_close - 1

second_yang_completed =
  current_bar_return >= 80% * first_yang_return
  AND current_close > current_open
  AND current_close >= middle_upper_edge
```

每到第1、2、3个整理日，只计算“截至当前决策时点已经出现”的中间K，不预知它是不是最后一根。`0.6`、`0.4`、`0.5%`、2%和末阳80%完成度均为工程默认；“两个低点尽量接近”只作为语境推断的加分项，不作为老冯公布的阳阴阳硬条件。

### 7.5 上方空间否决

老冯明确说过：压力位附近的阳阴阳胜率低，目标要结合第一根阳线的等距预期和左侧前高。

在买入前冻结最近已确认压力：

```text
resistance_candidates =
  pivot_high.confirmed_at <= decision_time
  AND pivot_high.price > candidate_entry

resistance = 候选中价格最低者；同价时取最近发生者
```

没有可确认的压力时，只用等距目标；不能事后挑选压力。老冯还说阳阴阳最好远离明显支撑或压力位，本版用“目标至少2R”硬控上方压力；下方支撑距离只记录为回测特征，不在没有数据验证前擅自加硬线。

### 7.6 买入：实盘版与日线回测版

**原始执行方式**：买点必须等到中间整理K线的尾盘判断。原话没有提供一种能预知“哪根是最后一根”的方法，因此程序不得使用“最后一根有效整理K”这个未来条件。

**分钟数据执行版【工程默认】**：

```text
decision_time = 14:50
ATR_ref       = 前一交易日收盘时已知的ATR14
middle_upper_edge = max(middle_high, 从第1个整理日至当日14:50)

在第1至第3个整理日中，首次满足：
  middle_valid_so_far = true
  AND provisional_high >= previous_day_high
  AND 尚未出现第二根大阳
  AND last_price >= middle_upper_edge - 0.1*ATR_ref

则在14:50冻结：
  decision_at、first_yang_return、middle_low、stop、resistance、entry_limit

若2% <= (ask1-stop)/ask1 <= 6%，且ask1<=entry_limit：
  14:50按卖一价提交限价单；
只允许向下改价，不允许追过entry_limit；14:55仍未成交则取消。
```

`0.1ATR`、卖一限价和5分钟有效期均为工程默认。

**只有日线数据的保守回测版【工程默认】**：

```text
execution_variant = DAILY_NEXT_OPEN_PROXY
signal_day = 第1至第3个整理日中，首次在收盘满足middle_valid_so_far的日期

if -3% <= next_open/signal_close-1 <= 2%:
    entry = next_open * (1 + slippage)
else:
    cancel
```

大幅低开被老冯视为阳阴阳风险；`-3%`和`+2%`是工程默认。这个日线代理不是尾盘原策略的等价回测：若下一日就是第二根大阳，买入后受T+1限制不能当日卖出。两种执行版本必须分开统计，不能混成一条收益曲线。

### 7.7 止损和目标

老冯没有公布统一的阳阴阳硬止损价，以下为工程定义：

```text
stop = min(middle_low_asof_decision)
       - max(1 tick, 0.1*ATR_ref)

raw_target = candidate_entry * (1 + first_yang_return)
target     = raw_target                              # 无resistance
target     = min(raw_target, resistance - 1 tick)   # 有resistance

equal_move_entry_limit = 2*stop / (2-first_yang_return)
resistance_entry_limit = (resistance-1 tick+2*stop) / 3
risk_entry_limit       = stop / (1-6%)

entry_limit = min(equal_move_entry_limit,
                  resistance_entry_limit,  # 无resistance时忽略
                  risk_entry_limit)
```

成交后：

```text
R_price = entry - frozen_stop
target  = entry*(1+first_yang_return)                         # 无resistance
target  = min(target, frozen_resistance-1 tick)               # 有resistance

2% <= R_price/entry <= 6%
(target-entry)/R_price >= 2
```

所有字段在14:50决策时冻结；买入后出现的新低不能拿来下移止损，后来形成的压力位也不能改写原计划。目标按“首阳涨幅在第二段大致重复”实现，更接近老冯10%、14%等实例；用首阳实体或整段距离计算目标只能作为分开的敏感性回测版本。

到目标全部卖出。原始口径也允许卖一半等待后续阳线，但本版为保持单一规则，默认全部退出。

### 7.8 阳阴阳失效

**入场前否决 `PRE_ENTRY_INVALID`**：

- 尚未成交就进入第4个整理日。
- 已观察到的中间K任一收盘跌破首阳实体中点。
- 已观察到的中间低点跌破首阳低点超过工程容差。
- 板块从`MAINLINE`变为`EXIT`。
- 冻结压力位使潜在收益小于2R。
- 止损距离不在2%-6%工程风险带内。
- 第二根大阳已经完成后才识别出形态。

**持仓后退出 `POST_ENTRY_EXIT`**：

- `low<=frozen_stop`：盘中硬止损事件。
- 收盘跌破买入时冻结的整理区下沿：结构失效事件。
- 板块进入`EXIT`或原逻辑被可验证信息证伪：逻辑失效事件。
- 到第4个整理日收盘仍未出现目标阳：形态超时事件。

持仓后事件记录发生时点；受T+1限制时先锁定`EXIT_PENDING`，最早到下一可卖且可成交时点执行。

### 7.9 阳阴阳时间退出

**【工程默认】**：

- `sellable_from`为买入日的下一交易日。
- `held_sellable_days`从`sellable_from`开始，只计算已经完整收盘的可卖交易日；买入日记为0。
- 第2个可卖交易日收盘，买入后最高价仍未达到`entry+0.5R`：记录不及预期，下一可成交时点退出。
- 第3个可卖交易日收盘仍未出现第二段大阳或`close/entry-1<3%`：记录时间退出。
- 第5个可卖交易日收盘仍未退出：记录最长持有退出。

`2日0.5R`和5日上限是工程默认；3日3%来自老冯通用历史口径，并非阳阴阳专属原句。

## 8. 策略 C：冰点核心修复先手

### 8.1 模块定位与来源边界

`ICE_CORE_REPAIR`是本规格组合多条原话后使用的工程名称，不是老冯本人公布过的固定策略名。

**【原始口径】**可以确认的交易路径是：

```text
市场进入冰点
-> 只考虑事先选好的主线或强逻辑核心
-> 弱势时倾向尾盘，开盘冰点也可能是买点
-> 用较小仓位取得下一交易日先手
-> 修复、大阳或高潮时主动高抛
```

老冯明确说过“冰点时增加仓位”“尾盘冰点抄底增加下一个交易日的仓位，等待反弹”，也说过“有先手高抛才行”和“很多就是一日游，第二天大幅拉升就卖了”。

**不能从原话直接得到的内容**包括：先手仓的固定比例、分钟止跌阈值、次日固定涨幅卖点，以及究竟卖一半还是全部。这些必须标为工程默认并回测。

本策略不是：

- `MAIN_DOWN + ICE`中无条件抄底。
- 盘中看到某只股票上涨或大跌后临时加入名单。
- 买从未走强过的普通弱股。
- 越跌越加仓，或用“试错”给无规则交易找借口。
- 用修复日收盘结果倒推前一日买点。

老冯后期明确反对把交易成败解释成“试错”。因此正式术语是“先手仓”，不是“试错仓”。

### 8.2 状态机

```text
IDLE
-> ICE_ARMED
-> WATCHLIST_FROZEN
-> WAIT_LIVE_REPAIR
-> LIVE_REPAIR_CONFIRMED
-> SETUP_CONFIRMED
-> ORDER_PENDING
-> POSITION_OPEN
-> EXIT_PENDING
-> CLOSED
```

旁路状态：

```text
INVALIDATED
EXPIRED
NO_FILL
DATA_INSUFFICIENT
LOOKAHEAD_VIOLATION
```

转换规则：

1. `t-1`收盘为ICE且不是MAIN_DOWN，进入`ICE_ARMED`。
2. `t-1`收盘冻结合格板块、核心股和证据，进入`WATCHLIST_FROZEN`。
3. 买入日开盘进入`WAIT_LIVE_REPAIR`，只监控冻结名单。
4. 实时大盘与板块修复连续成立，进入`LIVE_REPAIR_CONFIRMED`。
5. 个股形成分钟止跌收复结构，才进入`SETUP_CONFIRMED`。
6. 冻结入场、止损、压力、目标和预期对象后下单。
7. 第一个有效退出事件锁定，直到首个可卖且可成交时点执行。

### 8.3 盘前市场硬门槛

```text
prior_emotion_state = ICE
prior_trend_state != MAIN_DOWN
account_negative_feedback = false
```

`EXTREME_ICE`首版只记录，不自动放宽仓位。是否排除或允许极小仓属于待回测参数。

老冯提到冰点且放巨量时可以期待修复。程序把`ice_day_market_amount / market_amount_ref`作为质量特征，不在验证前设成硬门槛。

### 8.4 ICE_LEADER_CANDIDATE板块

普通策略要求正式`MAINLINE`。策略C允许一种更窄的候选状态：冰点当天已经有容量、逻辑、强度和联动，只差持续天数确认。

```text
ICE_LEADER_CANDIDATE =
  sector_state_asof_prior_close = CANDIDATE
  AND sector_day_hit = true
  AND linkage_ok = true
  AND sector_amount >= dynamic_big_floor
  AND logic_confirmed = true
  AND logic_source_date <= prior_close
  AND logic_expire_date >= buy_day
```

- `WATCH/WEAKEN/EXIT`不允许。
- 已是`MAINLINE`的板块仍可运行原有策略；不要为了使用策略C而主动降级。
- 冰点候选最多保留1个板块，按涨幅排名、成交额排名依次打破并列。
- 板块层级和成分必须在`t-1`冻结，不能在修复日选择最有利的概念归属。

### 8.5 核心股盘前冻结

```text
stock IN frozen_watchlist_asof_prior_close
AND liquidity_mode = STRICT
AND recognition_pass = true
AND recognition_rank_in_sector <= 3
AND volatility_ok = true
AND close >= MA20
AND MA20 >= MA60
```

每个板块最多冻结3只，依次按辨识度分、板块内成交额排名和20日相对强度排序。

策略C不继承“距旧日线支撑不超过6%”的硬线。风险锚点改为买入日已经确认的分钟结构低点；没有明确分钟止损仍然不买。

这不是降低选股标准。股票必须在冰点前已经走强并进入核心名单，不能从冰点日跌幅榜临时选普通弱股。

### 8.6 实时大盘与板块修复门

以下均为**【未校准工程默认】**：

```text
market_live_repair =
  adv_ratio_now >= 20%
  AND adv_ratio_now - ice_day_adv_ratio >= 10个百分点
  AND limit_down_count_now <= 80% * ice_day_limit_down_count
  AND count(index_last > index_open) >= 2
  AND 连续两个5分钟快照成立
```

板块同时满足：

```text
sector_live_ok =
  intraday_return_rank_pct <= 20%
  AND intraday_amount_rank_pct <= 20%
  AND sector_adv_ratio >= max(50%, market_adv_ratio_now + 20个百分点)
  AND frozen_cores_above_vwap >= 2
  AND 连续两个5分钟快照成立
```

日内成交额只能与同一时间戳、同一板块层级比较，不能把14:00部分成交额与其他板块的全日成交额混排。

这些修复门是首版保守化处理。老冯原话允许冰点直接拿尾盘先手，但没有给出统一分钟止跌条件；在失败样本回测完成前，程序不启用纯`PANIC_BUY`。

### 8.7 分钟买点A：RECLAIM_15M

研究时段为10:30—14:30，以下均为**【未校准工程默认】**：

1. 已形成第一个确认的15分钟低点`L1`。
2. 后面形成第二个确认低点`L2`，且：

```text
L2 >= L1 - 0.1 * ATR_ref
```

3. `L2`后价格突破收复位：

```text
reclaim_level =
  max(session_vwap,
      L1之后最近一个已确认15分钟pivot_high)
```

4. 最近完成的5分钟K收在`reclaim_level`上方。
5. 当前价格不高于`reclaim_level + 0.1ATR_ref`。

该变体要求先止跌、再收复，不在VWAP下方直接接加速下跌。

### 8.8 分钟买点B：TAIL_HOLD

弱势尾盘是老冯原始偏好，但以下精确条件属于**【未校准工程默认】**：

```text
decision_time = 14:50
order_end_time = 14:55

最近3根完整5分钟K全部收在VWAP上方
AND 13:30以后未再创日内新低
AND close_location_in_day_range >= 75%
AND last_price >= session_high - 0.25*ATR_ref
AND last_price <= session_vwap + 0.50*ATR_ref
AND 存在已确认15分钟结构低点
```

仍在VWAP下方、没有更高低点、只因为跌幅大而买的`PANIC_BUY`首版禁用。这样可能漏掉成功的深水反弹，但可以避免用赢家样本合理化接飞刀。

`RECLAIM_15M`和`TAIL_HOLD`必须分开统计，不能用“任一满足”拼成一条更好看的收益曲线。

### 8.9 下单、止损和目标

```text
trigger = reclaim_level + 1 tick
stop = latest_confirmed_structure_low
       - max(1 tick, 0.1*ATR_ref)

R_price = planned_entry - stop
```

必须满足：

```text
2% <= R_price / planned_entry <= 6%

target = entry + 2R                        # 无压力
target = min(entry + 2R, resistance-tick) # 有压力

(target-entry)/R >= 2
```

压力不足2R则放弃，不能为了“冰点会修复”降低盈亏比。

- 使用突破触发限价单，最多成交到`trigger + 0.1ATR_ref`。
- 订单有效5分钟，每只股票每天只尝试一次。
- 不追过冻结上限，14:55后取消所有新买单。
- 止损、压力和目标都在下单前冻结，买后不得向下移动止损。

### 8.10 先手仓与组合风险

老冯给出的账户语境有两种：先高抛再冰点接回，或者原本仓位很轻、尾盘增加下一交易日仓位。程序分别记录，不把两者混为“无上限加仓”。

**【未校准工程默认】**：

```text
module_market_multiplier = 0.50
effective_risk_pct =
  base_account_risk_pct
  * module_market_multiplier
  * volatility_multiplier
```

再取以下上限的最小值：

- 单票市值不超过账户10%。
- 本模块总市值不超过账户20%。
- 同一板块最多实际持有2只。
- 本模块同板块初始总风险不超过账户0.5%。
- 若当日先高抛后接回，新增市值不得超过已释放市值。
- 若原账户轻仓，策略C新增总市值不超过账户10%。

ETF与同主题股票即使以后同时支持，也必须归入同一风险簇。

### 8.11 次日修复与强势卖出

原始口径支持：低吸取得先手后，次日修复、大阳、涨停或高潮时主动兑现，允许卖飞。

首版继续使用买前冻结的2R和压力位，不新增“涨8%、10%、14%就卖”的固定数字，以免拟合四张截图。

退出事件按发生时间排序：

```text
HARD_STOP
LOGIC_INVALIDATED
REPAIR_THESIS_FAIL
TAKE_PROFIT
RELATIVE_FAIL
FAST_TIME_FAIL
THREE_DAY_FAIL
MAX_HOLD_EXIT
```

- `low <= frozen_stop`：记录硬止损。
- `high >= frozen_target`：首次可卖时全部止盈。
- 板块高潮或个股大阳可以记录`STRONG_SELL_WINDOW`，但未到冻结目标时首版只提示，不改写正式止盈价。
- 买入日触发止损也只能锁定事件，受T+1限制次日执行。
- 日线同时触及止损和目标且无分钟数据时，保守按止损先发生。
- 第3个可卖日累计不足3%退出；最长仍为5个可卖交易日。

`STRONG_SELL_WINDOW`是否允许先卖50%，以及剩余仓位何时卖，必须单独回测后才能变成自动订单。

### 8.12 策略C专属“不及预期”

买入时冻结：

```yaml
expectation:
  expected_event: 冰点后市场修复、候选板块强化、核心股快速反馈
  expected_deadline_sellable_days: 1
  min_progress_r: 0.5
  must_hold_price: frozen_reclaim_level
  hard_stop: frozen_structure_stop
  required_sector_states: [CANDIDATE, MAINLINE]
  relative_benchmark: frozen_sector_id
```

**结构失败**：买入后收盘重新跌破`frozen_reclaim_level`，记录退出事件，下一可成交时点执行。

**修复逻辑失败**：买入日或持仓日收盘，以下至少两项成立，记录`REPAIR_THESIS_FAIL`：

```text
市场实时修复门已经失效
板块不再满足维持条件
个股收盘跌破must_hold_price
```

**快速反馈失败【工程默认】**：第1个完整可卖交易日收盘，买入后最高价仍未达到`entry + 0.5R`，下一可成交时点退出。

**核心掉队【研究字段】**：板块仍位于涨幅前20%，个股却位于板块涨幅后50%。首版只记录，待回测后决定是否硬退出。

### 8.13 日线代理与缺数处理

策略C的原始买点包含开盘冰点、盘中获利盘砸盘和尾盘先手。仅靠日线无法知道买前是否止跌、收复VWAP，也无法确定买入后何时触发止损。

因此首版没有收益回测用的日线代理：

```text
只有日线
-> 可以输出盘前ICE_ARMED候选
-> 不生成成交
-> result = UNVERIFIABLE_WITHOUT_MINUTE_DATA
```

若以后增加`DAILY_NEXT_OPEN_PROXY`，必须作为独立实验。它不能与分钟版合并，也不能用来宣称复现了老冯的实际尾盘买入。

### 8.14 防未来函数断言

- ICE、趋势、板块状态、逻辑和核心名单全部截止`t-1 15:00`。
- 买入日临时进入榜单的股票不能补进冻结名单。
- 动态成交额基准只能截止`t-1`。
- 实时宽度、板块排名和VWAP只能使用`decision_time`前的数据。
- 分钟pivot必须等右侧K线完成后才确认。
- 止损不能使用决策后出现的日内最低价。
- 日内成交额排名只能比较同一时间戳。
- 买入日出现失败事件必须遵守T+1，不能假设当天卖出。

推荐原因码：

```text
REJECT_NO_PRIOR_ICE
REJECT_MAIN_DOWN
REJECT_SECTOR_NOT_PRELOCKED
REJECT_STOCK_NOT_PRELOCKED
WAIT_LIVE_REPAIR
REJECT_LIVE_SECTOR
SETUP_NOT_CONFIRMED
LOOKAHEAD_VIOLATION
DATA_INSUFFICIENT
NO_FILL
```

### 8.15 当前启用状态

```text
enabled = false
paper_only = true
calibration_status = UNCALIBRATED
```

先把规则写进系统，目的是开始收集完整样本，不是立即产生实盘信号。补齐分钟数据、失败样本和样本外回测前，程序只能输出候选和拒绝原因。

## 9. ETF独立通道

### 9.1 为什么不能混入股票池

ETF没有“板块个股前三”、个股涨跌停归一和公司逻辑字段。把ETF与股票一起排名，会扭曲成交额和辨识度。

v0.3只研究境内股票型行业/主题ETF。货币、债券、商品、跨境、杠杆和反向产品全部排除。

### 9.2 数据与复权

```text
raw_price, amount, vol <- fund_daily
factor                 <- fund_adj

adj_px[d|s] = raw_px[d] * factor[d] / factor[s]
```

- 技术指标使用`adj_px`。
- 真实成交、止损和目标使用当时未复权可成交价格。
- 成交额和成交量绝不复权。
- 拆分生效日若复权因子尚未可用，返回`CORPORATE_ACTION_QUARANTINE`，不生成信号。

连续性断言：

```text
raw_close[d-1] * factor[d-1] / factor[d]
≈ pre_close[d]
```

超出价格最小单位和数据容差时，返回`ADJUSTMENT_CONTINUITY_FAIL`。

### 9.3 ETF流动性与同类优选

ETF不套股票6亿/8亿元历史线。首版在境内股票ETF内部比较：

```text
median_amount_20_rank_pct <= 20%
AND current_completed_amount_rank_pct <= 20%
```

盘中只能比较同一时点累计成交额。20%是**【未校准工程默认】**。

同一跟踪指数有多个ETF时，依次选择：

1. 20日成交额更高。
2. 基金规模更大。
3. 历史跟踪误差更小。

### 9.4 跟踪板块与策略路由

必须保存带有效期的映射：

```text
index_code -> sector_id -> valid_from / valid_to
```

ETF成交额不计入股票板块容量，ETF也不增加板块活跃核心数量。

v0.3首版只允许ETF进入策略C：

```text
instrument_type = ETF
-> etf_universe_pass
-> tracked_sector = ICE_LEADER_CANDIDATE
-> ICE_CORE_REPAIR_ETF
```

简单回调和阳阴阳的ETF适配需另行验证，首版不自动复用。

### 9.5 折溢价与风险簇

折溢价必须使用决策时点可得的实时IOPV或交易所/券商数据。不能拿次日净值回填历史盘中溢价。

没有可靠实时IOPV时，折溢价字段记为`UNKNOWN`；首版允许研究记录，不据此自动下单。

ETF与其跟踪板块股票计入同一主题风险簇，共享策略C的模块20%市值上限和板块初始风险上限。

### 9.6 588170回归要求

588170于2026-07-03实施1:3份额拆分。回放必须先通过基金复权连续性断言；否则不得计算MA、ATR或把它与拆分前价格直接连接。

当前本地没有完整`fund_daily + fund_adj`和7月8日分钟数据，因此v0.3的正确结果是：

```text
instrument_route = ETF

eligibility = DATA_INSUFFICIENT
```

## 10. “不及预期”的程序定义

### 10.1 原始含义

老冯最直接的定义是：买入后即使上涨了，只要没有走出买入时的预期，也可以卖，因为原买入预期已经不存在。

另有明确时间规则：

```text
3 个交易日累计涨幅不足 3%，卖出。
```

这不是“亏了才算不及预期”，也不是“没立即涨停就卖”。它是交易前写好的预期路径失效。

### 10.2 每笔交易必须保存预期对象

```yaml
expectation:
  frozen_at: YYYY-MM-DD HH:MM:SS
  expected_deadline_sellable_days: 2
  min_progress_r: 0.5
  must_hold_price: 结构支撑或整理区间下沿
  required_sector_state: MAINLINE
  relative_benchmark: 所属板块指数
  expected_event: "回踩后反转/第二段大阳/催化兑现"
```

买入后不能修改这些字段来替亏损找理由。`2日、0.5R、必须维持MAINLINE`都是工程默认；原始口径只有“预期不存在就卖”和“3日没有3%则卖”。

### 10.3 硬失效

任一项满足即为 `HARD_NOT_EXPECTED`：

```text
close < must_hold_price
OR sector_state = EXIT
OR 买入逻辑的事件/业绩/政策被证伪
```

动作：下一可成交时点全部退出，不下移止损，不补仓摊薄。

`low<=hard_stop`单独产生盘中`HARD_STOP`事件，不重复归入收盘后才知道的`HARD_NOT_EXPECTED`。事件/业绩/政策只有在`result_known_at`之后才能判为证伪，不能用后来信息回填历史。

### 10.4 时间失效

```text
held_sellable_days =
  从sellable_from开始，已经完整收盘的可卖交易日数

time_fail =
  held_sellable_days >= expected_deadline_sellable_days
  AND max_high_since_entry < entry + min_progress_r*R_price
```

默认：2 个可卖交易日内连 0.5R 正反馈都没有，退出。

通用兜底：

```text
held_sellable_days >= 3
AND close / entry - 1 < 3%
```

`max_high_since_entry`只统计成交时点之后的数据。时间失效在收盘后才可知，因此只能生成下一可成交时点的退出单，不能假设按该日收盘价成交。

### 10.5 相对板块失效

高辨识度核心股应该至少不明显弱于所属主线。

**【工程默认】**：

```text
one_day_relative_fail =
  sector_ret >= 1%
  AND stock_ret <= -2%

two_day_relative_fail =
  sum(stock_ret-sector_ret, 2) <= -3%

relative_fail = one_day_relative_fail OR two_day_relative_fail
```

### 10.6 形态专属不及预期

除“3日不足3%”外，本节的日数、0.5R、板块相对强弱和事件窗口都是**工程默认**，必须单列敏感性回测。

**简单回调：**

- 反转信号买入后重新收回到信号 K 线低点之下。
- 2 个可卖交易日内没有突破信号 K 线高点。
- 板块走强而个股连续两日处于板块涨幅后 50%。

**阳阴阳：**

- 中间整理区下沿失守。
- 买入后 2 日没有出现至少 0.5R 的第二段上涨。
- 第 3 日仍没有第二根大阳，或累计涨幅不足 3%。

**事件催化：**

- 预定事件窗口结束后，板块没有联动。
- 个股没有出现放量或相对板块强度。
- 最迟事件次日收盘仍无反馈，下一交易日退出。

### 10.7 最终退出优先级

退出遵循“先发生、先锁定”，不是每天收盘后重新按原因覆盖：

```text
1. 所有退出事件保存 observed_at、price_level、sellable_at、executable_at。
2. 按 observed_at 排序；第一个有效事件锁定为 EXIT_PENDING。
3. 受T+1、跌停或停牌限制时保留该事件，直到可成交。
4. 只有事件时间完全相同，或日线无法判断同一根K内先后时，才使用保守类型优先级。

同时间优先级：
HARD_STOP > LOGIC_INVALIDATED > TAKE_PROFIT
          > RELATIVE_FAIL > TIME_FAIL > MAX_HOLD_EXIT
```

日线回测按以下时序：开盘前已有`EXIT_PENDING`先尝试成交；否则先处理开盘跳空，再处理盘中止损/止盈，最后在收盘后计算结构、相对强弱和时间失效。同一日`low<=stop`且`high>=target`又没有分钟线时，保守按止损先发生。

如果买入当日已经不及预期，受 T+1 限制，最早只能下一交易日卖出。回测不得假设当天买入、当天止损。

## 11. 账户与组合风险

### 11.1 以损定仓

```text
risk_budget = equity * effective_risk_pct
shares_by_risk = floor(risk_budget / R_price / 100) * 100

shares_by_position_cap =
  floor((equity * 20%) / entry / 100) * 100

shares = min(shares_by_risk,
             shares_by_position_cap,
             shares_by_remaining_exposure)
```

若结果小于 100 股，则不交易，不把风险预算向上取整。

### 11.2 同板块风险

**【工程默认】**：

- 同一板块最多同时持有 3 只。
- 同板块市值不超过账户 30%。
- 同板块初始总风险不超过账户 1.5%。
- 高相关板块按同一风险簇合并计算，例如 AI 算力与服务器不能完全视为独立风险。

### 11.3 账户负反馈

```text
account_negative_feedback =
  rolling_5d_return <= -3%
  OR drawdown_from_20d_high <= -5%
  OR 3笔连续按规则止损
```

动作：

- 总仓位上限降一级。
- 单笔风险减半。
- 连续两日账户回正且无新增规则违背后，才恢复一级。

以上阈值为工程默认；“账户优先、账户下跌减仓”是老冯原始原则。

## 13. 回测成交模型

### 13.1 禁止未来函数

- 当天收盘数据产生的信号只能在下一交易日成交。
- 阳阴阳第二根大阳完成后，不能反过来假设买在中间阴线。
- 摆动高低点需要右侧 K 线确认，确认前不可用。
- 历史板块成分、ST 状态、上市状态、涨跌停制度和逻辑字段必须 point-in-time。
- 不能用今天认定的龙头标签回填历史。
- 复权因子必须按当时可知口径处理，成交额使用未复权真实值。
- 策略C的ICE、板块候选和核心名单只能截止前一交易日收盘。
- 策略C的修复门、VWAP、分钟pivot和累计成交额只能截止决策时点。
- ETF拆分日若复权因子当时尚未可得，必须隔离，不能事后补因子产生当日信号。

### 13.2 无法成交

- 一字涨停且无可成交量：不假设可以买到。
- 一字跌停：不假设能按止损价卖出，订单延续到下一可成交时点。
- 停牌：订单顺延或按策略取消。
- 开盘跳空超过规则容许范围：取消新买单。

### 13.3 同日止盈止损歧义

只有日线数据且同一根 K 线同时触及目标和止损时：

```text
默认按止损先发生
```

这是保守假设。使用分钟线后按真实时间顺序处理。

### 13.4 成本

回测至少加入：

- 佣金。
- 卖出印花税，按历史时点税率。
- 过户费等实际费用。
- 买卖滑点。
- 100 股整数手。

费用和税率必须按历史日期配置，不能用当前费率覆盖全部历史。

## 16. v0.3 最终规则

如果只保留最核心的可编程条件：

```text
1. MAIN_DOWN 不开普通新仓，策略C同样禁用。
2. ICE 不是满仓信号；策略C只观察前一日已冻结的强逻辑核心。
3. 板块不做线 = 最近5日沪深成交额中位数 × 0.953143%；低于即排除。
4. 大板块线 = 最近5日沪深成交额中位数 × 1.571759%；达到后仍须有强度、联动、持续性和可追溯逻辑。
5. 个股流动性用LOOSE/STANDARD/STRICT三档；交易默认STANDARD = 市场额 × 0.068503%，并同时位于成交额前6.25%。
6. 简单回调只做日线上升趋势的双/三底反转；前低止损、2R且前高下方止盈。
7. 阳阴阳在首阳后的第1-3个整理日逐日尾盘判断，首次有效才允许下单；禁止预知“最后一根”，4根以上、压力太近、第二阳已完成都放弃。
8. 冰点核心修复只允许`t-1=ICE`、非主跌、板块为ICE_LEADER_CANDIDATE、个股为预先冻结的前3核心。
9. 策略C必须等实时修复和分钟止跌收复；只有日线时不模拟成交。
10. 策略C次日修复、大阳或高潮是强势卖出窗口，但正式自动止盈仍以买前冻结的2R和压力为准。
11. ETF独立复权、独立排名，只通过跟踪板块路由；拆分因子缺失时隔离。
12. 每笔交易买前冻结entry、stop、target、R、仓位和expectation。
13. 2日无0.5R反馈、3日不足3%、相对主线明显转弱，都算不及预期；策略C另有首个可卖日反馈检查。
14. T+1、涨跌停、停牌、滑点、基金拆分和无法成交必须进入回测。
```

本版参数的唯一正确用法是：先固定、再回测、再做滚动样本外验证。不能在看完整收益曲线后反复调到最好看。
